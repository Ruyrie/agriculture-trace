1. 项目定位

该项目是一个前后端分离的农产品溯源系统。业务核心不是单纯维护产品资料，而是围绕“产品、批次、生产记录、质检记录、物流记录、访问记录、数据指纹、链式审计日志”建立一套可查询、可验证、可监管的追溯平台。

用户可以登录系统管理产品和批次，录入生产/质检/物流信息；消费者或监管者可以进入溯源详情查看产品流转过程；管理员和监管员可以查看数据指纹、审计日志链和统计分析，从而判断数据是否可能被绕过系统篡改。

系统主题农产品从生产到流通的全过程数字化追踪。
业务对象产品、批次、生产记录、质检记录、物流记录、访问记录、用户、角色、审计日志。
关键增强使用 SHA-256 数据指纹和 previous_hash 链式日志模拟区块链不可篡改思想。
角色范围农户、监管员、管理员，不同角色看到不同菜单和接口权限。

5. 代码职责解析

代码区域作用解释为什么这样写
controller 接收 HTTP 请求，解析参数，调用 Service，统一返回 Result。 让接口层轻量，业务规则集中在 Service，便于维护和测试。
service 实现业务规则，例如产品 CRUD、批次唯一性、哈希计算、审计日志、溯源记录保存。 Service 能使用事务保证多表写入一致，例如新增批次和三类溯源记录要同生共死。
repository Spring Data JPA 仓库，处理产品、批次、用户、记录、日志的常规查询。 减少手写 CRUD SQL，方法名即可表达常见查询。
mapper/StatisticsMapper.xml 统计报表 SQL：分类分布、趋势、产地分布、批次产量、访问排行。 聚合 SQL 用 MyBatis 更直接，可读性比在 JPA 中拼装更高。
config/SecurityConfig.java 配置 Session 登录、remember-me、角色权限、统一 JSON 登录响应。 登录状态交给 Spring Security 管理，前端通过 Cookie 自动携带身份。
frontend/src/router/index.js 定义页面路由和角色可访问范围，并用 /user/info 校验 Session。 前端提供友好的页面拦截，后端继续做最终权限兜底。
frontend/src/utils/request.js 统一 Axios baseURL、Cookie、Result 拆包、401 清理和跳转。 页面不需要重复处理登录过期和后端响应结构。

农产品溯源系统安装指引

适用于 Spring Boot 后端 + Vue 3 前端 + MySQL 8 的本地开发环境部署。

1. 项目打开路径 2. 环境要求 3. MySQL 配置 4. IDEA 模块配置 5. 启动后端 6. 启动前端 7. 登录账号 8. 常见问题

1. IDEA 要打开哪个路径

请用 IntelliJ IDEA 打开项目根目录下的 class01 文件夹，而不是打开外层的 agriculture-trace。

D:\6.12\agriculture-trace\class01

打开后项目结构应类似下面这样：

class01/
  backend/      后端 Spring Boot 模块
  frontend/     前端 Vue 3 + Vite 模块
  README.md
  install-guide.html
如果 IDEA 打开的是 D:\6.12\agriculture-trace，可能只看到外层文件，前后端模块识别会比较混乱。建议关闭项目后重新选择 class01 目录打开。

2. 环境要求

软件 推荐版本 说明
JDK 17 或更高 后端基于 Spring Boot 3，需要 JDK 17。
Maven 3.8+ 用于下载后端依赖和启动后端。
Node.js 18 或更高 用于安装前端依赖和启动 Vite。
MySQL 8.0+ 保存用户、产品、批次、溯源记录、哈希和审计日志。
IntelliJ IDEA Ultimate 或 Community 后端推荐用 IDEA 打开，前端可以用 IDEA 的 Terminal 执行命令。

可以在命令行检查版本：

java -version
mvn -version
node -v
npm -v
mysql --version

3. MySQL 配置

3.1 启动 MySQL 服务

先确保 MySQL 已启动，并且能用客户端或命令行连接。

mysql -uroot -p

3.2 默认数据库配置

后端默认读取 backend/src/main/resources/application.yml 中的配置：

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/agriculture_trace?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true&createDatabaseIfNotExist=true
    username: root
    password: 123456

默认含义：

  • 数据库地址：localhost:3306
  • 数据库名：agriculture_trace
  • MySQL 用户名：root
  • MySQL 密码：123456

3.3 首次启动会自动建库建表

项目的 JDBC URL 带有 createDatabaseIfNotExist=true，并且 spring.sql.init.mode=always。正常情况下，后端启动时会自动创建数据库并执行：

backend/src/main/resources/schema.sql

该脚本会创建表、初始化角色、用户、产品、批次、溯源记录、数据指纹和审计日志。

3.4 如果 MySQL 账号或密码不是 root / 123456

有两种改法，推荐第一种。

方式 A：修改 application.yml

打开文件：

D:\6.12\agriculture-trace\class01\backend\src\main\resources\application.yml

把下面两项改成你自己的 MySQL 账号和密码：

spring:
  datasource:
    username: 你的MySQL账号
    password: 你的MySQL密码

方式 B：使用环境变量覆盖

不改代码文件，直接在启动后端前设置环境变量：

MYSQL_USERNAME=你的MySQL账号
MYSQL_PASSWORD=你的MySQL密码

如果端口、地址或数据库名也不一样，还可以设置：

MYSQL_URL=jdbc:mysql://localhost:3306/agriculture_trace?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true&createDatabaseIfNotExist=true
如果你的 MySQL 用户没有创建数据库权限，请先手动创建数据库：CREATE DATABASE agriculture_trace DEFAULT CHARACTER SET utf8mb4;，然后再启动后端。

4. IDEA 前后端模块配置

4.1 打开项目

  1. 打开 IntelliJ IDEA。
  2. 选择 File > Open。
  3. 选择路径 D:\6.12\agriculture-trace\class01。
  4. 等待 IDEA 扫描项目。

4.2 配置后端 Maven 模块

  1. 在 IDEA 左侧找到 backend/pom.xml。
  2. 右键 pom.xml，选择 Add as Maven Project。
  3. 等待 Maven 下载依赖。
  4. 进入 File > Project Structure > Project，确认 Project SDK 是 JDK 17。
  5. 如果 backend/src/main/java 没有被识别为源码目录，右键该目录，选择 Mark Directory as > Sources Root。
  6. 如果 backend/src/main/resources 没有被识别，右键该目录，选择 Mark Directory as > Resources Root。

4.3 配置前端模块

  1. 前端目录是 frontend。
  2. 前端不需要 Maven，使用 Node.js 和 npm。
  3. 可以直接在 IDEA 底部 Terminal 里进入 frontend 执行命令。
  4. 如果 IDEA 提示配置 Node interpreter，选择你本机安装的 Node.js。
简单理解：backend 是 Maven/Spring Boot 模块，frontend 是 Node/Vite 模块。两个模块都在同一个 class01 项目目录下。

5. 后端安装与启动

5.1 命令行启动

在 class01 目录下打开终端，执行：

cd backend
mvn clean install
mvn spring-boot:run

看到类似下面的信息，说明后端启动成功：

Tomcat started on port 8080
Started AgricultureTraceApplication

后端默认地址：

http://localhost:8080

5.2 在 IDEA 中启动

  1. 打开 backend/src/main/java/com/example/agriculturetrace/AgricultureTraceApplication.java。
  2. 点击类旁边或 main 方法旁边的绿色运行按钮。
  3. 选择 Run AgricultureTraceApplication。

5.3 后端测试命令

cd backend
mvn test

6. 前端安装与启动

6.1 安装依赖

第一次运行前端必须先安装依赖：

cd frontend
npm install

6.2 启动前端开发服务器

npm run dev

正常启动后会出现前端地址，一般是：

http://localhost:5173

6.3 前端接口地址配置

前端开发环境接口地址在：

D:\6.12\agriculture-trace\class01\frontend\.env.development

默认内容：

VITE_API_BASE_URL=http://localhost:8080

如果后端端口不是 8080，需要把这里改成后端实际地址，例如：

VITE_API_BASE_URL=http://localhost:8090

6.4 前端打包验证

cd frontend
npm run build

8. 常见问题

8.1 后端连接 MySQL 失败

先检查三件事：

  1. MySQL 服务是否启动。
  2. application.yml 里的用户名和密码是否正确。
  3. MySQL 用户是否有创建数据库和建表权限。

可以用命令验证账号密码：

mysql -u你的账号 -p你的密码

8.2 端口被占用

后端默认端口是 8080。如果被占用，修改：

backend/src/main/resources/application.yml

把端口改成其他值：

server:
  port: 8090

同时修改前端 .env.development：

VITE_API_BASE_URL=http://localhost:8090

8.3 前端 npm install 失败

建议先确认 Node.js 版本：

node -v
npm -v

如果网络较慢，可以使用 npm 镜像：

npm config set registry https://registry.npmmirror.com
npm install

8.4 页面能打开但接口报错

通常是前端接口地址和后端实际地址不一致。检查：

  • 后端是否已经启动。
  • 后端端口是否是 8080。
  • frontend/.env.development 是否指向正确后端地址。

8.5 修改配置后没有生效

修改后端 application.yml 后需要重启后端。修改前端 .env.development 后需要重启前端开发服务器。

推荐启动顺序：先启动 MySQL，再启动后端，最后启动前端。
